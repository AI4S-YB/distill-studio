# 本地配置与密钥管理

`distill-studio` 目前支持两种密钥使用方式：

- GUI 中直接填写 API key
- CLI / Shell 中通过环境变量传入

另外，如果你要在本机执行正式 release 打包，还可能会多一个本地签名密码：

- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`

## GUI 配置如何保存

GUI 中的 `API key` 会保存在本地配置档案里，界面默认以密码形式隐藏显示。

开发模式下，本地 profile 默认位置：

```text
config/local/profiles/
```

发布版默认位置：

- macOS: `~/Library/Application Support/com.ai4s.distillstudio/`
- Linux: `~/.config/com.ai4s.distillstudio/`
- Windows: `%APPDATA%\\com.ai4s.distillstudio\\`

说明：

- 发布版不会再依赖项目源码目录中的 `config/local/`
- 这样可以避免安装包或发布仓库暴露开发机信息

## 运行输出中的密钥处理

流水线运行时会把生成参数写入 `generate_config.json`，但真实 `api_key` 会被主动清空，不会落盘到输出目录。

这能避免：

- 把运行结果目录直接打包后带出真实密钥
- 把输出目录上传到远程服务器时顺手泄露密钥

## 推荐的本地习惯

- GUI 开发：使用本地 profile 保存 key
- CLI 调试：使用 `.env.local` 或 shell 环境变量
- 本地 release 打包：把 `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` 放到 `.env.local` 或当前 shell
- 对外示例：只保留 `.example` 文件，不保留真实 key

说明：

- 这个密码只用于 `npm run tauri:build:release`
- 普通开发模式 `npm run tauri:dev` 不需要它
- 只有当你的 `~/.tauri/distill-studio.key` 本身是加密私钥时才需要填写

## 不应提交到 GitHub 的内容

- `.env.local`
- `config/local/gui.pipeline.json`
- `config/local/profiles/*.json`
- 任何包含真实 `sk-...` 的文件

## 示例文件

可以对外保留的示例文件：

- `.env.local.example`
- `config/examples/generate_config.qwen.example.json`
- `config/examples/generate_config.qwen_canary.example.json`
- `config/examples/default.profile.example.json`
