# 测试问题收口

本文用于汇总当前版本对测试问题的处理状态，便于回归验证和发版说明。

状态说明：

- `已解决`：功能已经落地，当前版本可直接验证
- `部分解决`：已有明显改进，但和原始期望相比还有边界或取舍
- `不再处理`：当前阶段不继续投入，通常通过产品策略规避

## 一、安装测试

### 问题1

Rust 工具链安装时要注意，Windows 上编译需要 Visual Studio C++ 工具链（MSVC v143、Windows 11 SDK）。

- 状态：`已解决`
- 说明：
  - 已在文档中明确写出 Windows 开发环境前置要求
  - 相关文档：
    - `docs/Windows安装与启动.md`
    - `README.md`

### 问题2

启动 GUI 开发模式原先需要 Bash / WSL / Git Bash，并手动 `source ./.env.local`。

- 状态：`已解决`
- 说明：
  - `npm run tauri:dev` 现在会自动读取 `.env.local`
  - 不再要求 Bash 的 `source`
  - PowerShell、CMD、Git Bash 都可以直接运行
  - 相关脚本：
    - `scripts/run-tauri-with-env.mjs`

### 问题3

Windows 全新环境首次启动 Tauri 桌面应用耗时很长，需要下载大量 crates。

- 状态：`不再处理`
- 说明：
  - 这是开发模式冷启动成本，不是业务逻辑 bug
  - 当前策略是不再继续优化开发机首启，而是通过“最终用户使用安装包”规避
  - 文档中已经明确说明首次冷启动慢是正常现象

## 二、使用测试

### 问题1

生成 QA 时没有进度条，缺少 `已完成 shard / 总 shard 数` 和 shard 内进度。

- 状态：`已解决`
- 说明：
  - 已增加总进度条
  - 已显示 shard 总进度
  - 已显示当前 shard 内进度和总体生成进度

### 问题2

QA 生成页面无法直接暂停或终止，只能在终端终止。

- 状态：`部分解决`
- 说明：
  - 运行按钮现在会切换为“停止”
  - 可直接在 GUI 中停止当前任务
  - 当前仍不是“暂停后继续当前 shard”，而是“停止”

### 问题3

不能查看运行日志内容，希望日志区可滚动并支持导出。

- 状态：`已解决`
- 说明：
  - 日志区已支持滚动
  - 已增加“导出日志”按钮

### 问题4

`shard_0000.json` 已经生成，但【浏览QA】页面没有内容。

- 状态：`已解决`
- 说明：
  - 浏览页现在会优先读取：
    - `dataset.jsonl`
    - `pack_summary.json`
  - 如果还没 pack，也会回退读取：
    - `generated/shard_*.json`
  - 因此中途生成的批次也能被浏览

### 问题5

设置页面参数修改没有提示，运行中参数被锁定但没有提示。

- 状态：`已解决`
- 说明：
  - 已增加运行中锁定提示
  - 已增加设置页和 QA 生成页的前置提示卡
  - 运行时参数锁定状态现在可见

### 问题6

“取消”和“添加所选标签”按钮顺序希望调整为“添加所选标签”在右侧。

- 状态：`已解决`
- 说明：
  - 当前按钮顺序已经调整

### 问题7

没有历史任务记录入口。

- 状态：`已解决`
- 说明：
  - 已提供“浏览QA”入口
  - 已改为历史任务批次列表
  - 可查看任务时间、主题、结果数量、模型等摘要信息

### 问题8

输出文件没有快捷入口，希望完成后可直接打开输出文件夹。

- 状态：`已解决`
- 说明：
  - QA 生成页已增加“打开输出文件夹”按钮

### 问题9

没有运行统计面板，希望显示 ETA、成功率、失败数、平均速度、token 消耗等。

- 状态：`部分解决`
- 说明：
  - 已增加运行统计面板
  - 当前已支持：
    - 已运行时间
    - 平均速度
    - 当前速度
    - ETA
    - 生成进度
    - 请求进度
    - 分片进度
    - 重试次数
    - 失败请求
    - 请求成功率
  - `token` 消耗统计当前决定不做

### 问题10

“续跑已有 shard”选项没有起作用，重新运行同一任务会新建目录，不能自然续跑。

- 状态：`部分解决`
- 说明：
  - 已支持：
    - 继续当前任务
    - 从历史任务精确继续
    - `resume-batch`
    - 自动复用最近一次匹配任务目录
  - QA 生成页现在也能直接下拉选择历史任务继续，不必先切到浏览页
  - 但当前仍不是“手动选择任意本地目录”模式

### 问题11

任务完成后没有结果统计信息。

- 状态：`已解决`
- 说明：
  - 完成后已有结果摘要卡
  - 当前会显示：
    - 生成数量
    - 保留数量
    - shard 数
    - request 数
    - 输出目录
    - 数据集路径

## 当前剩余结论

当前最主要的未完全闭合项只剩两类：

1. `Windows 开发模式首次冷启动慢`
   - 当前不继续投入优化
   - 通过安装包交付规避

2. `续跑能力仍不是手动选择任意目录`
   - 现有方案已经覆盖大多数实际续跑需求
   - 如果后续用户仍明确要求“选择任意旧目录”，再单独补目录选择器
