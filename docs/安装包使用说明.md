# 安装包使用说明

这份文档面向最终用户。

如果你拿到的是 `QA小灶` 的 Windows 安装包，那么：

- 不需要安装 Rust
- 不需要安装 Node.js
- 不需要安装 Visual Studio C++ 工具链
- 不需要运行 `npm run tauri:dev`
- 不需要接触源码目录、`.env.local`、Git Bash、WSL

你只需要像安装普通软件一样完成安装，然后按界面引导做一次配置。

## 1. 下载与安装

通常你会拿到以下其中一种文件：

- `DistillStudio_x.y.z_x64-setup.exe`
- `DistillStudio_x.y.z_x64_en-US.msi`

安装步骤：

1. 双击安装包
2. 按安装向导完成安装
3. 安装完成后，从桌面快捷方式或开始菜单打开 `QA小灶`

说明：

- 第一次安装不需要开发环境
- 安装路径一般使用默认值即可
- 普通用户不需要自己配置系统级开发工具链
- 安装包文件名使用 ASCII 安全名 `DistillStudio`，应用界面内仍显示中文名称 `QA小灶`

## 2. 第一次打开后需要做什么

首次打开后，通常只需要做一次模型配置。

当前程序在第一次启动时会弹出一张欢迎提示卡，直接告诉你：

- 先去 `设置`
- 填一次模型厂商、模型和 API key
- 再回到 `QA生成` 填主题并点击 `运行`

当前版本的第一档方案仍然是“客户端直连模型服务”，所以最终用户至少还需要：

- 选择模型厂商
- 确认模型名称
- 填写对应平台的 API key

填好后，程序会把这些配置保存在本地。只要不卸载程序或手动清理配置目录，后续一般不需要重复填写。

## 3. 用户实际使用流程

普通用户的实际使用顺序通常是：

1. 打开 `QA小灶`
2. 在 `设置` 页完成一次模型配置
3. 回到 `QA生成`
4. 输入研究主题
5. 点击 `运行`
6. 在 `浏览QA` 中查看历史任务和生成结果

## 4. 自动更新

如果发布版已经接通自动更新，普通用户后续一般不需要手动下载安装新版本。

典型流程是：

1. 在左侧点击 `检查更新`
2. 如果有新版本，程序会提示是否安装
3. 确认后自动下载、安装并重启

如果提示“当前已是最新版本”，说明不需要处理。

## 5. 卸载后会发生什么

卸载程序时，一般需要分两部分理解：

- 程序本体会被卸载
- 本地配置和历史数据是否删除，取决于卸载器和用户是否额外清理数据目录

当前发布思路下，程序配置和工作区数据默认放在系统目录，而不是源码目录。

这意味着：

- 普通用户不会接触项目文件夹
- 更新程序时通常不会丢失已有配置

## 6. 常见问题

### 1. 为什么安装包用户不需要 Rust / Node / Visual Studio

因为这些是“从源码开发和编译”时才需要的工具。

最终用户使用的是已经构建好的桌面应用安装包，不需要再本地编译。

### 2. 为什么安装完成后还要填 API key

因为当前第一档方案仍然是客户端直连第三方模型服务。

也就是说：

- 安装过程可以做到像普通软件一样
- 但模型服务鉴权仍然需要用户自己提供一次 API key

### 3. 安装完成后第一次运行较慢正常吗

如果只是安装包用户，第一次打开通常不会像开发模式那样慢很多。

但首次创建本地配置、初始化工作区、读取模型配置时，仍可能有短暂等待，这是正常现象。

### 4. 最终用户和开发者的区别是什么

最终用户：

- 使用安装包
- 不看源码
- 不跑 `npm run tauri:dev`
- 不需要 Rust / Node / C++ 工具链

开发者：

- 从源码启动
- 需要 Node、Rust、Tauri、Windows C++ 工具链
- 会接触 `.env.local`、`npm run tauri:dev` 这类命令

## 7. 给最终用户的一句话说明

如果你要给普通用户一段最短说明，可以直接发这段：

```text
下载安装包，像普通软件一样安装。第一次打开后，在“设置”里填写一次模型 API key，后续就可以直接使用，不需要安装 Rust、Node 或其它开发环境。
```
